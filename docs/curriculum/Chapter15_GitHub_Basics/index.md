> **Goal:** take Phase 9's local Git skills and put them on a real remote — fluent in repos, issues, PRs, forks, releases, projects, discussions, branch protection, and CODEOWNERS.

## What you build

A polished public `rag-notes` repo on GitHub with:

- A clean README + LICENSE + comprehensive `.gitignore`.
- A `feature/embedding` branch shipped via a reviewed, squash-merged PR.
- 8 standardised labels (`bug`, `feature`, `docs`, `good first issue`, …) and 2 milestones.
- Structured **issue forms** (`bug_report.yml`, `feature_request.yml`) with dropdowns and required fields.
- An open issue, a closed-by-PR issue, and a release `v0.1.0` with auto-generated notes.
- A Discussions tab with a welcome thread.
- A Project (v2) tracking issues across columns with auto-move workflows.
- Branch protection on `main` (reviews, status checks, linear history).
- CODEOWNERS routing PRs to the right teams automatically.
